# GEMS Video AI

## Overview
GEMS Video AI is a full-stack AI SaaS platform designed for generating ultra-realistic cinematic videos from various inputs like stories, scripts, or images. The platform supports video lengths from 10 seconds to 3 hours and offers seven distinct video styles. It aims to provide a comprehensive solution for video creation with advanced AI capabilities, including deep analysis for production planning and a sophisticated pronunciation system for natural-sounding dialogue. The project is built on a pnpm monorepo architecture and includes robust features for user management, real-time analytics, and an integrated ad system. The business vision is to revolutionize video content creation by making high-quality, AI-generated cinematic videos accessible.

## System Architecture

### UI/UX Decisions
The frontend utilizes React, Vite, Tailwind CSS, Framer Motion, and shadcn/ui for a modern and responsive user interface. Wouter is used for routing. The application features a user profile dashboard with dedicated tabs for profile management, video history, and subscription plans. An admin panel provides comprehensive platform management with six tabs: Users, Videos, Languages, Ads, Analytics, and System. The design includes status badges, progress bars, and real-time data displays. Non-intrusive ad placements are integrated into the dashboard and galleries.

### Technical Implementations
The platform is built on a pnpm monorepo using Node.js 24 and TypeScript 5.9. The backend is powered by Express 5, with PostgreSQL and Drizzle ORM for data persistence. Zod is used for schema validation. Authentication is handled via JWT with role-based access control (user, admin, super_admin). API codegen is done using Orval from an OpenAPI specification. The build process uses esbuild.

### Feature Specifications
- **Video Generation**: Generates cinematic videos from stories, scripts, or images with customizable styles and durations.
- **Role-Based Access Control (RBAC)**: Granular permissions for users, admins, and super_admins, including exclusive access to certain admin features.
- **Authentication**: JWT-based authentication with token storage in `localStorage`.
- **User Management**: Admin panel for managing user statuses (active, blocked, terminated) with immediate enforcement.
- **Real-time Analytics**: Dashboard for platform statistics, user activity, render pipeline status, and ad performance, updated every 5 seconds. Additionally, a full **Analytics Dashboard** (super_admin only) with 4 sub-tabs: Users (registration trends, role/status distribution, top creators, recharts charts), Revenue (total/MRR/daily trend, per-language breakdown, regional split), Videos (style/status/resolution/duration distributions, cinema mode rate, creation trend), and GPU (utilization, queue depth, worker type, daily throughput). Data is served by 4 new dedicated backend endpoints (`GET /admin/analytics/users|revenue|videos|gpu`).
- **Maintenance Mode**: Super-admin configurable maintenance mode with custom messages.
- **User Profile Dashboard**: Allows users to manage their profile, view video history, and control subscriptions.
- **Smart Ad System**: Full monetization ad system with a global ON/OFF platform toggle (super_admin only, defaults OFF). When enabled, ads are served only to free users — admin/super_admin and VIP/Special Access users are always 100% ad-free (enforced at API level via special_access check). Placements: sidebar (compact AdUnit), dashboard gallery (full AdUnit), and top banner. Admin "Ads" tab provides: global toggle with live status indicator, per-ad CRUD (title, description, type image/video/banner, placement, status, media URL, target URL, CTA text, scheduled start, expiry date), impression/click/CTR analytics with 14-day daily chart. Impression and click events are recorded per-user in `ad_events` table. The `GET /api/ads` endpoint gate-checks the `ads_enabled` platform setting and user exemption before returning any ads.
- **High-Speed Rendering**: Priority-based, GPU-accelerated parallel rendering pipeline with distinct tiers (Enterprise, Paid, Free) offering varying scene counts, parallel workers, and render times. Includes real-time scene progress, queue position, and ETA countdown.
- **AI Core Engine**: Utilizes OpenAI `gpt-5.2` (via Replit integration) for deep AI analysis of videos, generating production plans including story intelligence, character profiles, emotional arcs, cultural profiles (specifically Indian), scene breakdowns, and production blueprints.
- **Pronunciation System**: Ensures natural-sounding voices by preventing mispronunciations through a global language dictionary and voice correction mechanisms. Includes AI script analysis for IPA, flagged words, and confidence scores, with admin and user interfaces for managing and contributing to the dictionary and voice corrections.
- **Audio + Realism System**: Integrates audio profiles (voice science, performance directives, lip-sync criticals) and realism profiles (human truth statements, artifact prevention) into the video generation workflow. Includes background music cues and emotion timelines.
- **API Platform**: Super-admin controlled system for managing API clients and keys, including generation, status management, and analytics.
- **Offers / Promotions System**: Super-admin can create discount offers (percentage or fixed) with optional coupon codes, validity windows, max-use limits, and applicable plan filtering. Users redeem codes via `/offers/redeem`. Redemption is atomic (DB-enforced unique `(offer_id, user_id)` and conditional `used_count` increment) so concurrent redemptions cannot exceed limits or duplicate per user. Codes are normalized to UPPERCASE on both create and redeem. Admin tab shows live/expired/exhausted status, top-performing offers, and full enable/disable/edit/delete controls.
- **Special Access (VIP) System**: Super-admin can grant per-user privileges that override the normal plan: free video generation, unlimited usage, all-languages access, premium features unlock. Each grant has a reason (influencer/testing/promotion/vip/other), optional expiry, optional notes, and an active toggle. The unique `(user_id)` index ensures one entry per user. `/me/special-access` returns the user's live status (honoring expiry + active flag). Admin "VIP Access" tab includes a tracking dashboard with total/active/inactive counts, total usage actions, estimated revenue impact, breakdown by reason, and top VIP users.
- **Sound Prompt Validation + Auto-Replacement**: Detects sound cues inside scripts and validates each cue against the scene's emotion, environment, and intensity. Decisions are KEPT (good cinematic match), REPLACED (weak/generic prompt swapped for a curated copyright-free library sound), or ADDED (missing ambient/environmental layer suggested). Powered by `gpt-5.2` with strict JSON output. The library is admin-managed (CRUD with name, category, tags, emotions, environments, intensity, audio URL, license, attribution) and matched to scenes via a deterministic scoring helper. Users access at `/sound-design`; super-admins manage the library in the "Sound Lib" admin tab and view stats (total/active entries, analyses, decision splits, top replacement sounds).
- **Paranormal & Supernatural VFX Intelligence (Auto)**: Automatically runs on every new video — no UI, no button, fully transparent. After the deep analysis completes, the Paranormal Engine silently scans the script/story for horror, paranormal, supernatural, and superhero keywords and context (bhoot/ghost/demon/flying/tantrik/possessed/superpower etc.). It generates a structured profile: content type classification, character ability profiles (power level, evolution arc, VFX keywords, physics notes), per-element VFX requirements (technique, color grading, avoid list), atmosphere profile (lighting, color palette, sound design), physics rules for fictional elements, style directives (camera rules, hero moments, signature shots), cultural blend (Indian tantrik/haveli/mythology + global Hollywood), and strict realism directives. This profile is stored in `paranormal_profiles` and automatically injected into the screenplay generation prompt — so any screenplay involving supernatural elements automatically gets correct VFX instructions, physics rules, and atmosphere directives without the user doing anything. Everything is cinematic-realistic (no cartoon look).
- **Cinema-Grade Output System**: Every new video includes a full Cinema Output configuration: aspect ratio (16:9 / 9:16 / 21:9 / 2.39:1 anamorphic), resolution (1080p / 2K / 4K / 8K), auto-activated Cinema Mode at 4K+ (injects theatrical mastering directives into AI screenplay generation), color grade (Natural / Warm / Cold / Noir / Vibrant / Indian Vivid), film grain toggle (35mm analog), depth of field toggle (rack focus/bokeh), audio mastering (Stereo or Surround-Ready 5.1/7.1), and multi-select export formats (YouTube, Instagram, DCP, ProRes, RAW, HLS). All settings are stored in the `videos` DB table and injected into the screenplay and audio generation prompts via a `CinemaSpec` directive block. The New Video form has an interactive Cinema Output card with visual aspect ratio boxes, cinema-labeled resolution buttons, gradient color swatches, and toggle switches. The video detail page shows cinema spec badges (resolution, aspect, color grade, DoF, grain, surround, export targets) below the standard status badges.
- **Long Video Split + Intermission System**: Automatically handles feature-length films exceeding standard durations. Videos over 1.5 hours (5400s) receive a **cinematic intermission**: the screenplay AI inserts a dedicated INTERMISSION scene at the natural story midpoint, with a fade-to-black card in the film's primary language (Hindi: "ANTRAKTA", Tamil: "INTERVAL", etc.), soft bridging music, and a re-anchoring scene after the break. Videos over 3 hours (10800s) also receive a **2-part split**: the screenplay AI marks a `part1_end` / `part2_begin` boundary at an act/chapter boundary (never mid-dialogue), generates Part 2 opening slates, a narrative continuity note, and Part 1/Part 2 closing/opening slates. After screenplay generation, the AI engine scans scenes, calculates cumulative timing, and persists `hasIntermission`, `intermissionAtSecond`, `intermissionLabel`, `isSplit`, `splitAtSecond`, `part1Title`, and `part2Title` to the DB. The video detail page shows a "Long Film Structure" card with the intermission timing and per-part download buttons.
- **Auto Marketing Assets Generation**: Automatically generates promotional assets from completed videos. Logic: videos ≥1 hour receive a cinematic trailer (1–2 min script/storyboard with scenes, hooks, music cues, transitions), a film-style promotional poster image, and a YouTube thumbnail; videos <1 hour receive a thumbnail only. All text/title in the same language as the video. Assets are stored in the `marketing_assets` DB table and generated asynchronously (fire-and-forget, 202 response, then client polls). Users trigger generation from the "Marketing Assets" panel on the video detail page, with controls to regenerate or delete individual assets. Poster and thumbnail are AI-generated PNG images stored as base64 data URIs.

- **Security Hardening**: Production-grade security layer in `artifacts/api-server/src/middlewares/security.ts`. Includes: Helmet HTTP headers (CSP, HSTS, X-Frame-Options, etc.), tiered rate limiting (auth: 10 req/15 min, video creation: 20 req/10 min, admin: 100 req/15 min, general: 500 req/15 min), anti-bot middleware (blocks empty/bot User-Agents, filters suspicious path traversal patterns), JWT algorithm pinning (HS256 enforced on sign + verify), and content moderation hook (calls OpenAI Moderation API on video creation prompts; skips gracefully if key absent).

### System Design Choices
The `artifacts` directory houses the `api-server` (Express backend) and `gems-video-ai` (React frontend). The `lib` directory contains shared resources like `api-spec` (OpenAPI source of truth), `api-client-react` (generated hooks), `api-zod` (generated schemas), `db` (Drizzle ORM setup), and `integrations-openai-ai-server`. The `requireAuth` middleware performs live DB status checks on every request to ensure immediate enforcement of user status.
